# Download Queue Pause Semantics (等待 vs 暂停)

## 背景

用户反馈：首次导入后任务全部显示为“已暂停”，但按百度网盘逻辑，未开始的任务应显示“等待中”。“暂停全部”只应影响正在下载的任务，未开始的任务保持等待。

## 目标

- 任务状态 `status`
  仅表示任务真实状态（pending/downloading/paused/failed/...）。
- `queue_paused` 仅作为“调度暂停”开关，不映射为任务暂停。
- 导入或启动时，未开始的任务显示为“等待中”。
- “暂停全部”只暂停下载中的任务，不改变 pending。

## 设计选择

采用方案 B：保持 `queue_paused`
为调度开关，前端依然以任务状态为准显示。避免把 pending 任务转成 paused。

## 变更要点

- 后端在文件存在时不再将 pending 任务强制改为 paused。
- 保留已暂停任务的 paused 状态（仅当用户显式暂停）。
- 任务进度/已下载大小仍会被刷新，便于断点续传。

## 测试

新增单测：当磁盘上已有部分文件时，任务依旧保持 pending 状态。
